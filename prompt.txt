📚 Project Documentation:

Please include all Markdown files from the repository (e.g. `BUILD_SCRIPT_ARCHITECTURE.md`, `IMPLEMENTATION_GUIDE.md`, `SECURITY.md`, `CLAUDE.md`, etc.) in your prompt context and decision-making. These files define the structure, safety principles, flow architecture, optimization strategies, and AI agent integration of AILinux.

They serve as the authoritative knowledge base for:
- modular task execution
- system security and GPG policy
- Claude swarm coordination and memory persistence
- performance-aware build logic
- fallback and emergency recovery features

Ziel: Erstelle ein vollständiges, robustes `build.sh`-Skript zur Erstellung einer bootfähigen Live-ISO für AILinux, basierend auf Ubuntu 24.04 mit KDE Plasma, Calamares Installer, Secure Boot, AI-Terminal-Helfer (`aihelp`) und integriertem apt-Mirror.

Das Skript soll folgende Anforderungen erfüllen:

1. **KDE 6.3 Desktop-Umgebung** mit minimalem Setup  
2. **Calamares Installer** mit Branding und funktionierendem Bootloader (UEFI + BIOS)  
3. **Secure Boot Support** (shimx64.efi.signed)  
4. **AILinux Mirror-Integration** (`archive.ailinux.me`)  
5. **AI-Integration über Mixtral/OpenAI API** über `/opt/ailinux/aihelp`  
6. **GPG-Signatur des Repos** mit eigenem Keyring  
7. **Fehlerrobustes Mount-/Unmount-Handling**  
8. **Funktionierender Live-Boot, Offline-Verwendung**  
9. **Verzeichnisstruktur wie im GitHub-Repo `https://github.com/derleiti/ailinux-beta-iso`**  

⚠ **Aktueller Bug:**  
Das `build.sh`-Skript loggt den Benutzer während der Ausführung *aus* (vermutlich durch Neustart von Systemdiensten, chroot-Probleme oder aggressive Bereinigung). Bitte identifiziere und behebe die Ursache, sodass das Skript zuverlässig im Terminal durchläuft, ohne dass die Session beendet wird.

Verwende best practices für bash scripting, Logging und Fehlerbehandlung. Die Ausgabe soll auch remote per SSH nutzbar sein.

Nutze nötige Kommentare und prüfe, ob `set -eo pipefail` ggf. zu aggressiv ist.

➕ **Erweiterung – Prüfsummen berücksichtigen:**  
Im Repository befinden sich mehrere `*.md5`-Dateien, die Prüfsummen wichtiger Build-Artefakte (z. B. ISO, EFI-Dateien, Kernel-Module) enthalten. Das Skript soll:

- beim Erstellen neuer Dateien automatisch prüfen, ob eine passende `.md5` existiert  
- bei Abweichungen optional eine neue `.md5` erzeugen  
- einen optionalen Hook wie `validate-md5.sh` nutzen, um vor dem finalen ISO-Schritt alle relevanten Checksummen zu verifizieren  
- die MD5-Validierung sauber in die Swarm-Koordination einbinden (Fehler, Erfolg, Logs)

Ziel: Ein konsistenter Build-Prozess mit validierten Artefakten und automatischer Checksummenprüfung.
