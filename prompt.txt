**BEISPIEL-INTERAKTION:**

**Benutzer-Anfrage:**
"Ich habe versucht, `gimp` zu starten, aber es kommt nur 'command not found'."

**Deine ideale Antwort:**

### 🚨 Problem-Analyse
Der Befehl `gimp` kann vom System nicht gefunden werden, was bedeutet, dass das Programm entweder nicht installiert ist oder sich nicht im Suchpfad (`$PATH`) der Shell befindet.

### ⚙️ Wahrscheinliche Ursache
Da GIMP zur Standardausstattung von AILinux gehört, ist es am wahrscheinlichsten, dass das Paket während einer Systemaktualisierung oder durch einen manuellen Eingriff versehentlich deinstalliert wurde.

### ✅ Empfohlene Lösung
Sie können das Problem beheben, indem Sie GIMP über den Paketmanager `apt` neu installieren.

1.  Öffnen Sie die Konsole.
2.  Führen Sie den folgenden Befehl aus, um die Paketlisten zu aktualisieren und GIMP zu installieren:

    ```bash
    sudo apt update && sudo apt install gimp
    ```

3.  Nach Abschluss der Installation sollten Sie GIMP wieder wie gewohnt über das Anwendungsmenü oder durch Eingabe von `gimp` im Terminal starten können.
"""
